ABSTRACT 
 
In the modern digital era, software systems generate a massive volume of log data, which plays a crucial role in monitoring, debugging, and ensuring system reliability. However, the unstructured and high-dimensional nature of these logs makes manual inspection time-consuming, error-prone, and inefficient. To address these challenges, this project proposes a Hybrid Log Classification System that integrates both rule-based techniques (Regex) and machine learning models (BERT and DeepSeek-R1 LLM) to achieve accurate and scalable log classification.The system architecture begins with the ingestion of raw log messages through a FastAPI-based endpoint, followed by preprocessing steps such as tokenization, cleaning, and pattern extraction. Simple and repetitive log patterns are effectively handled using lightweight Regex-based classification, ensuring efficiency in detecting known errors. For complex, contextual, and unseen log patterns, the system leverages BERT embeddings combined with the DeepSeek-R1 large language model, enabling semantic understanding and contextual classification beyond simple keyword matching.By combining traditional deterministic methods with deep learning-based approaches, the hybrid framework ensures both high precision and adaptability to evolving log structures. The pipeline is designed for scalability.  deep learning models in terms of accuracy, robustness, and efficiency.This project contributes a comprehensive end-to-end log analysis solution that minimizes manual intervention, reduces system downtime, and accelerates root-cause identification. Beyond classification, the system’s modular design allows future extensions into log anomaly detection, predictive maintenance, and automated incident response. In conclusion, the proposed Hybrid Log Classification System exemplifies how Generative AI, NLP, and traditional rule-based methods can be synergistically integrated to solve real-world operational challenges in software engineering.

CHAPTER 1 
INTRODUCTION 
 
1.1 BACKGROUND AND CONTEXT 
In modern software systems, large volumes of log data are continuously generated from multiple sources such as servers, applications, APIs, and monitoring tools. These logs play a crucial role in understanding system behavior, diagnosing failures, and ensuring system reliability. However, due to the increasing scale and complexity of distributed systems, manual log analysis has become inefficient, time-consuming, and error-prone.
Traditional log analysis techniques rely heavily on rule-based systems and manual inspection, which are not scalable in real-world environments. Logs are often unstructured and contain diverse patterns, making it difficult to extract meaningful insights using conventional methods.
With the advancement of Artificial Intelligence (AI) and Natural Language Processing (NLP), log messages can now be treated as textual data. Deep learning models such as BERT have shown remarkable capability in understanding contextual relationships within text. This enables automated systems to analyze logs more effectively by capturing semantic meaning rather than relying only on predefined patterns.Therefore, intelligent log classification systems are becoming essential for modern IT infrastructure to improve monitoring, anomaly detection, and system maintenance.
 
1.2 RESEARCH PROBLEM AND SIGNIFICANCE 
Despite advancements in log analysis, several challenges still exist:
* Log data is highly unstructured and dynamic 
* Traditional methods depend on manual rules and parsing 
* Difficulty in detecting unknown or complex patterns 
* Large-scale logs require high computational efficiency 
* Existing systems either focus only on machine learning or deep learning, but fail to combine multiple techniques effectively.
  

Fig: 1.1. The Challenge of Manual Log Analysis
To overcome these limitations, there is a need for a hybrid system that integrates:
* Semantic understanding (BERT embeddings) 
* Pattern recognition (Regex rules) 
* Unsupervised learning (Clustering) 
Such a system can significantly improve classification accuracy, reduce manual effort, and enhance system reliability.


1.3 RESEARCH OBJECTIVES: 
The main objectives of this project are:
* To develop an automated system for log classification 
* To apply BERT-based embeddings for semantic understanding 
* To perform clustering for grouping similar log messages 
* To integrate regex-based rules for pattern recognition 
* To classify logs into predefined categories 
* To evaluate system performance using metrics such as accuracy, precision, recall, and F1-score.




1.4 METHODOLOGY OVERVIEW 
The proposed system follows a hybrid approach combining deep learning, machine learning, and rule-based techniques.Initially, log messages are preprocessed and converted into vector representations using BERT embeddings. These embeddings capture semantic relationships between log messages.Next, clustering techniques are applied to group similar logs. Regex-based rules are then used to refine classification by identifying known patterns.
Finally, the system assigns labels such as:
* HTTP Status 
* Critical Error 
* Security Alert 
* System Notification 
* Resource Usage 








































CHAPTER 2  
LITERATURE SURVEY 
 
 
2.1 INFERENCES FROM LITERATURE SURVEY 
Over the past decade, the rapid growth of distributed systems, cloud computing, and large-scale applications has led to an exponential increase in system-generated log data. Log files contain valuable information about system behavior, errors, user activities, and performance metrics. Efficient analysis of such logs is essential for system monitoring, anomaly detection, and fault diagnosis. Researchers have proposed various machine learning and deep learning techniques to address these challenges. The following section presents a detailed analysis of existing works in log anomaly detection.
1. He, P. et al. conducted a comprehensive evaluation study on log anomaly detection techniques by comparing both traditional machine learning methods and deep learning approaches. Their research focused on understanding how different models perform under varying log structures and datasets. Traditional methods such as Support Vector Machines and clustering algorithms were found to be effective for structured logs but struggled with unstructured and high-dimensional data. On the other hand, deep learning models demonstrated improved performance by capturing hidden patterns and dependencies within logs. However, the study concluded that no single model can handle all types of log data effectively, highlighting the need for adaptive and hybrid approaches.
2. Du, M. et al. proposed DeepLog, a pioneering deep learning-based framework that utilizes Long Short-Term Memory (LSTM) networks to model log sequences. In this approach, system logs are treated as sequences similar to natural language sentences. The LSTM model learns normal execution patterns during training and detects anomalies by identifying deviations from these learned patterns. DeepLog is particularly effective in capturing temporal dependencies between log events, making it suitable for sequential anomaly detection. Despite its high accuracy, the model requires a large amount of labeled training data and significant computational resources, which can limit its scalability in real-time applications.
3. Zhang, X. et al. introduced LogRobust, an advanced anomaly detection model based on attention-enhanced Bi-LSTM networks. The key advantage of this model is its ability to handle unseen log messages by learning semantic relationships between log tokens. The attention mechanism allows the model to focus on important parts of the log sequence, improving detection accuracy. This makes LogRobust more robust compared to earlier models. However, the system relies heavily on log parsing techniques to convert raw logs into structured templates. Any change in log format can negatively impact the parsing process, thereby reducing the effectiveness of the model.
4. Le, V. and Zhang, H. proposed a novel approach for log anomaly detection that eliminates the dependency on log parsing. Instead of converting logs into structured templates, their model directly processes raw log messages using deep learning techniques. This approach improves flexibility and makes the system adaptable to dynamic log formats. It reduces preprocessing overhead and simplifies the pipeline. However, processing raw logs increases computational complexity and requires more memory and processing power, making it less efficient for large-scale systems.
5. Guo, H. et al. developed LogBERT, a transformer-based model inspired by Bidirectional Encoder Representations from Transformers (BERT). This model leverages self-attention mechanisms to understand contextual relationships between log messages. LogBERT is capable of capturing both local and global dependencies within log sequences, leading to highly accurate anomaly detection. The use of masked language modeling further enhances its ability to detect abnormal patterns. However, transformer-based models are computationally expensive and require GPU acceleration, which increases deployment cost and limits real-time applicability.
6. Brown, T. B. et al. introduced large-scale language models capable of performing tasks using few-shot and zero-shot learning techniques. These models can generalize well even with minimal labeled data, making them useful for handling rare or unseen log patterns. Their adaptability makes them suitable for dynamic environments where log data continuously evolves. However, these models are not specifically designed for log analysis tasks, and their large size results in high computational overhead, which can be impractical for production systems.
7. Ester, M. et al. proposed the DBSCAN (Density-Based Spatial Clustering of Applications with Noise) algorithm, which is widely used for clustering and anomaly detection. In the context of log analysis, DBSCAN groups similar log messages into clusters and identifies anomalies as noise points that do not belong to any cluster. The algorithm is effective in detecting irregular patterns without requiring labeled data. However, it is highly sensitive to parameter selection such as epsilon and minimum points, which can significantly affect clustering performance.
8. Devlin, J. et al. introduced BERT, a deep bidirectional transformer model that has revolutionized natural language processing tasks. BERT provides contextual embeddings by analyzing words based on their surrounding context in both directions. In log analysis, BERT can be used to generate meaningful representations of log messages, improving anomaly detection accuracy. Despite its effectiveness, BERT requires extensive computational resources and long training times, making it challenging for continuous monitoring systems.
9. Landauer, T. K. and Dumais, S. T. proposed Latent Semantic Analysis (LSA), a classical technique for representing textual data in a lower-dimensional vector space. LSA captures relationships between words and documents using matrix decomposition methods. In log analysis, it helps in identifying patterns and similarities between log messages. However, LSA lacks the ability to capture deep contextual meaning and has been largely replaced by more advanced models such as word embeddings and transformers.
10. Bertero, C. et al. conducted a survey on natural language processing techniques for log analysis. The study explored methods such as TF-IDF, LSA, and recurrent neural networks for extracting information from log data. It provided a strong foundation for early log analysis systems and highlighted key challenges in preprocessing and feature extraction. However, the survey does not include modern deep learning models such as transformers, limiting its relevance for current research.

Unlike traditional methods such as TF-IDF, embeddings provide deeper understanding by considering the context of words. This allows the system to differentiate between similar log messages based on meaning rather than keywords.

5.7 CLUSTERING OF LOG MESSAGES:
Clustering is an unsupervised learning technique used to group similar log messages based on their feature representations. In the proposed system, clustering plays a crucial role in identifying patterns and detecting anomalies in log data.
After generating embeddings for log messages, clustering algorithms such as DBSCAN (Density-Based Spatial Clustering of Applications with Noise) are applied. DBSCAN is particularly suitable for log analysis because it can identify clusters of arbitrary shapes and detect noise points (outliers) without requiring a predefined number of clusters.
The clustering process involves grouping log messages based on their similarity in the embedding space. Logs that belong to the same cluster share similar patterns, while logs that do not belong to any cluster are considered anomalies.

5.8 LOG CLASSIFICATION:
Log classification is the process of assigning predefined labels to log messages based on their content. In the proposed system, a hybrid classification approach is used, combining machine learning and deep learning techniques.
The classification process involves two main approaches:
1. Machine Learning-Based Classification
Traditional machine learning algorithms such as Random Forest, Support Vector Machine (SVM), and Logistic Regression are used to classify log messages based on embedding features. These models are trained using labeled data and learn patterns associated with each category.
2. Deep Learning-Based Classification
A BERT-based classifier is used to improve semantic understanding of log messages. BERT analyzes the context of words and captures relationships between different parts of the log message, resulting in more accurate classification.
The hybrid approach combines the strengths of both methods, ensuring better performance and robustness.
Classification categories include:
• HTTP Status
• Critical Error
• Security Alert
• Error
• System Notification
• Resource Usage
• User Action
• Workflow Error
• Deprecation Warning
The classification model learns from training data and predicts labels for new log messages, reducing manual effort and improving system efficiency.

5.11 PERFORMANCE EVALUATION:
The performance of the proposed log classification system is evaluated using standard machine learning metrics such as accuracy, precision, recall, and F1-score. These metrics provide a comprehensive understanding of the system’s effectiveness in classifying log messages correctly.
Accuracy represents the overall correctness of the model, indicating the percentage of correctly classified log messages. Precision measures how many of the predicted positive instances are actually correct, while recall indicates how many actual positive instances are correctly identified. The F1-score is the harmonic mean of precision and recall, providing a balanced evaluation of the model. The results indicate that the system achieves high accuracy across multiple categories such as HTTP Status, Security Alert, and System Notification.
The high precision and recall values demonstrate that the model is capable of accurately identifying log categories with minimal false positives and false negatives. The F1-score further confirms the robustness of the system.
Additionally, the model shows consistent performance across different classes, indicating that it is not biased toward any specific category. This ensures reliability in real-world applications.

CHAPTER 7 
CONCLUSION 
 
In conclusion, this project presented a hybrid log classification system that integrates multiple techniques including regex-based preprocessing, embedding generation using sentence transformers, clustering, deep learning-based classification, and Large Language Model (LLM)-based fallback mechanisms. The primary objective of this system is to efficiently analyze and classify large-scale and heterogeneous log data generated from various enterprise systems such as CRM, Billing, HR, Analytics, and Legacy systems. The hybrid architecture plays a crucial role in improving both accuracy and efficiency by combining the strengths of different approaches.
The use of regex-based techniques provides fast and efficient classification for structured log messages, reducing computational overhead in the initial stages. The embedding model (all-MiniLM-L6-v2) captures semantic relationships within log messages, enabling the system to understand contextual meaning rather than relying solely on keywords. This significantly improves the classification of complex and unstructured logs. Additionally, clustering techniques such as DBSCAN help in grouping similar log patterns, enhancing pattern discovery and reducing redundancy in data processing.
The results obtained from the system demonstrate an overall accuracy of approximately 99%, indicating the effectiveness of the proposed approach. The system successfully classifies log messages into categories such as Security Alert, HTTP Status, Workflow Error, System Notification, and Deprecation Warning. The API-based implementation using FastAPI enables real-time log classification, making the system suitable for practical deployment in enterprise applications.
Overall, the proposed hybrid log classification system provides a scalable, flexible, and efficient solution for automated log analysis. It reduces manual effort, improves anomaly detection, and enhances system monitoring capabilities. Future work can focus on integrating real-time streaming platforms, optimizing model performance, implementing automated retraining mechanisms, and enhancing visualization tools.
